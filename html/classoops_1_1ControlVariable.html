Detailed Description

template<typename MODEL>
class oops::ControlVariable< MODEL >

Control Variable.

The control variable acts as a container for the inputs of the variational data assimilation cost functions in physical space. That includes the states at the start the assimilation window or of each sub-window but also additional variables such as model bias, VarBC coefficients, or other control variables for algorithms that use them. This is mostly a convenience class that is used to keep things together and reduce the number of arguments to be passed around.

The control variable \( x \) has the following structure

\[ x = \begin{bmatrix} x_{state4D} \\ x_{modaux} \\ x_{obsaux} \end{bmatrix} \]

where

\[ x_{state4D} = \begin{bmatrix} x^0 \\ x^1 \\ .. \\ x^N \end{bmatrix} \]

and \( N \) is the number of sub-windows.

-> \( x_{state4D} \) is implemented by State4D containing a vector of State objects

\[ x_{modaux} = \begin{bmatrix} x_{modaux}^0 \end{bmatrix} \]

-> \( x_{modaux} \) is implemented by ModelAuxControl

\[ x_{obsaux} = \begin{bmatrix} x_{obsaux}^0 \\ x_{obsaux}^1 \\ .. \\ x_{obsaux}^K \end{bmatrix} \]

-> \( x_{obsaux} \) is implemented by ObsAuxControls which contains a vector of ObsAuxControl objects ( \( x_{obsaux}^i \)), which in turns contain a map of ObsAuxControlBase objects (e.g \( x_{VarBC}, x_{TOVSCV} \)):

\[ x_{obsaux}^i = \begin{bmatrix} x_{obsaux,i}^0 \\ .. \\ x_{obsaux,i}^L \end{bmatrix} \]

-> IFS: K=0, L=1:

\[ x_{obsaux}= \begin{bmatrix} \begin{bmatrix} x_{VarBC} \\ x_{TOVSCV} \end{bmatrix} \end{bmatrix} \]
