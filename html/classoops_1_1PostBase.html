Detailed Description

template<typename FLDS>
class oops::PostBase< FLDS >

Handles post-processing of model fields.

PostBase is the base class for all state post processors, it is mostly used so that PostProcessor can hold a vector of such processors.

By default processing is performed on every call.

PostBase is templated on FLDS; FLDS would typically be either State or Increment.

Post-processing is organized in three stages/methods: initialize (called prior to time-stepping), process (called during the forecast at each time step) and finalize (called once the time-stepping is finished).

Methods initialize, process and finalize wrap around virtual methods (implemented by PostBase children classes) doInitialize, doProcessing and doFinalize.

PostBase class contains a PostTimer object. PostTimer class controls the frequency of post-processing.

PostBase is the base class of the following classes (refer to their documentation for more information):

  • Observer
  • StateInfo
  • StateSaver
  • StateWriter
  • TrajectorySaver
  • WeightedDiff
  • WeightedMean
  • JqTerm
