Detailed Description

template<typename MODEL>
class oops::CostJo< MODEL >

\( J_o \) term of the cost function

The CostJo class encapsulates the \( J_o \) term of the cost function. It inherits from a pure abstract base class CostTermBase which defines a common interface to all the cost function terms except for the \( J_b \) term.

The \( J_o \) term has the following structure:

\[ J_o = \frac{1}{2} [y - {\cal G}(x)]^T R^{-1} [y - {\cal G}(x)] \]

\[ J_o = \sum_l J_o^l = \frac{1}{2} \sum_l [\bar{y}_{l} - {\cal \bar{G}}_{l}(\bar{x}_{l})]^T \bar{R}_{l}^{-1} [\bar{y}_{l} - {\cal \bar{G}}_{l}(\bar{x}_{l})] = \frac{1}{2} \sum_j \sum_k \sum_l [ \bar{y}_{k,l}^j - {\cal \bar{G}}_{k,l}^j (\bar{x}_{k,l}^j)]^T [\bar{R}_{k,l}^j]^{-1} [ \bar{y}_{k,l}^j - {\cal \bar{G}}_{k,l}^j (\bar{x}_{k,l}^j) ] \]

-> Loop over l - loop over observation types;

-> Loop over k - loop over the time steps spanning the assimilation window/sub-window;

-> Loop over j - loop over sub-windows;

Note: the loop over the number of sub-windows is external to CostJo.

Note: that the \( J_o \) class contains objects, which are themselves containers to allow exposing to OOPS multiple observation types (look k above):

-> ObservationSpaces -> a vector of ObservationSpace objects;

-> ObsOperators -> a vector of ObsOperator objects;

-> LinearObsOperators -> a vector of LinearObsOperator objects;

-> ObsErrors -> a vector of ObsErrorCovariance objects;

-> Observations -> a vector of ControlObsVector objects;

-> Departures -> a vector of ControlObsVector objects;

Note:

-> \( {\cal G}(x) \) is computed by the Observer post-processor (attached to the non-linear forecast);

-> \( y \) is implemented by Observations class;

-> \( y - {\cal G}(x) \) is stored by Departures class;

-> \( R \) is implemented by ObsErrors class;

In more detail, the \( J_o \) term includes the constrained observation bias component (constrained VarBC); consider \( J_o^l \) corresponding to the l-th observation space (observation type):

\[ J_o^l = \frac{1}{2} (\bar{y}_l - {\cal \bar{G}}_l (\bar{x}_l))^T [\bar{R}_l]^{-1} (\bar{y}_l - {\cal \bar{G}}_l (\bar{x}_l)) = \frac{1}{2} (\begin{bmatrix} \tilde{y}_l \\ \tilde{b}_l \end{bmatrix} - \begin{bmatrix} {\cal \tilde{G}}_l && {\cal \tilde{P}}_l \\ 0 && {\cal \tilde{P}}_l \end{bmatrix} \begin{bmatrix} x \\ \tilde{\beta}_l \end{bmatrix})^T \begin{bmatrix} \tilde{R}_l^{-1} && 0 \\ 0 && \tilde{R}_{\beta,l}^{-1} \end{bmatrix} (\begin{bmatrix} \tilde{y}_l \\ \tilde{b}_l \end{bmatrix} - \begin{bmatrix} {\cal \tilde{G}}_l && {\cal \tilde{P}}_l \\ 0 && {\cal \tilde{P}}_l \end{bmatrix} \begin{bmatrix} x \\ \tilde{\beta}_l \end{bmatrix})^T \]

-> the non-linear observation operator associated with l-th ObservationSpace

\[ \begin{bmatrix} {\cal \tilde{H}}_l && {\cal \tilde{P}}_l \end{bmatrix} \]

is implemented by obsEquiv method of ObsOperator; note that \( {\cal \tilde{G}}_l = {\cal \tilde{H}}_l {\cal I} \), where \( {\cal I} \) is an interpolation operator

-> the non-linear observation bias opeartor \( {\cal \tilde{P}}_l \) associated with l-th Observation space is is implemented by obsBiasEquiv method of ObsOperator;

-> the observation and bias error covariance matrix associated with l-th ObservationSpace:

\[ \begin{bmatrix} \tilde{R}_l^{-1} && 0 \\ 0 && \tilde{R}_{\beta,l}^{-1} \end{bmatrix} \]

is implemented by inverseMultiply method of ObsErrorCovariance class; \( \tilde{R}_l \) is observation error covariance matrix and \( \tilde{R}_{\beta,l}\) is observation bias error covariance matrix;

-> the control observation vector associated with l-th ObservationSpace: 

\f[ \begin{bmatrix} \tilde{y}_l \\
                    \tilde{b}_l
    \end{bmatrix}
\f]

is implemented by ControlObsVector class. Observations class contains a
vector of ControlObsVector objects;

-> The departures associated with l-th ObservationSpace: 

\f[ \begin{bmatrix} \tilde{d}_l \\
                    \tilde{d}_{c,l}
    \end{bmatrix} =
    \begin{bmatrix} \tilde{y}_l \\
                    \tilde{b}_l
    \end{bmatrix}
  - \begin{bmatrix} {\cal \tilde{G}}_l && {\cal \tilde{P}}_; \\
                      0                && {\cal \tilde{P}}_l
    \end{bmatrix}
    \begin{bmatrix} x \\
                    \beta_l
    \end{bmatrix}
\f]

are implemented by ControlObsVector class. Departures class contains a
vector of ControlObsVector objects.

For details on the implementation of VarQC refere to ObsErrorCovariance documentation.

Consider the quadratic \( J_o \) cost function term:

\[ J_o(dx) = G^T R^{-1} G (dx - d) \]

-> \( G (dx) \) is computed by ObserverTL; ObserverTL is a post processor which is attached to the tangent linear model trajectory (LinearModel::forecastTL). \( G \) has the same structure as \( {\cal G} \) and \( dx \) has the same structure as \( x \);

-> \( G^T (dy) \) is computed by ObserverAD; ObserverAD is a post processor which is attached to the adjoint of the tangent linear model trajectory (LinearModel::forecastAD).

template<typename MODEL >
double oops::CostJo< MODEL >::finalize ( const eckit::Configuration &  conf) const
overridevirtual

Various computations after the integration of the model.

This method is called after the non-linear model integration. Its purpose is to compute departures, perform screening if required, set up the observation error covariance matrix, VarQC weights, print Jo table, perform diagnostics if required. The Jo cost value is returned.

  1. Retrieve observation equivalents \( {\cal G}(x) \)

AROME: this is required for radar observations

  1. Calculate departures \( d = y - {\cal G}(x) \)
  2. Write the observation space component of the first guess IFS: this is required to write out the VarBC cycle file
  3. Screen observations
  4. Linearize the observation error covariance matrix R
  5. Setup VarQC weights
  6. Compute FG gradient \( R^{-1}d \)
  7. Print JO table
  8. Diagnostics
  9. Reset the Observer

Implements oops::CostTermBase< MODEL >.

Various set ups prior to the integration of the model.

This method is called prior to the non-linear model integration. Its purpose is to construct the observation operator, observation error covariance matrix, read observations. It returns an Observer to be attached to the non-linear trajectory; the purpose of the Observer is to compute model equivalents at observation locations.

  1. At each outer loop setup a switch to control if screening is on;
  2. Store a reference to the first guess
  3. Reset observation space if required
  4. Construct \(\cal H\), R, y if required
  5. Set up the Observer
  6. Return the Observer

template<typename MODEL >
void oops::CostJo< MODEL >::screenObservations ( std::unique_ptr< Observations_ > &  yeqv,
const eckit::Configuration &  conf 
) const

Screen observations.

This method is called to perform screening of observations. Once the screening is completed, the observation operator, observation error covariance matrix, observations, departures and observation equivalents are reset.

  1. Print unscreened Jo table (set up of observation error covariance matrix and VarQC weights is required)
  2. Save departures in the current ObservationSpace
  3. Get a vector of pointers to GeoVaLs from the Observer
  4. Perform screening - this saves a screened ObservationSpaces
  5. Close the current ObservationSpaces and read in the screened ObservationSpaces
  6. Reset the observation error covariance matrix \(R\)
  7. Reset the non-linear observation operator \(\cal G\)
  8. Reset observations
  9. Reset departures
  10. Compute observation equivalents as \( {\cal G}(x) = y - d \)
  11. Reset the observation space component of the first guess
  12. Reset the observation space component of the background
