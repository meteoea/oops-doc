23 \section how_to_build_execs How to build the executables
24 
25 Under linux, use version 4.4.2 or later of gcc and gfortran.

35 
36 Then, to build the executables:
37 
38  - (cd src; gmake)
39  - (cd $model; gmake)
40 
41 where <tt>$model</tt> determines which model you will be using.
42 Currently, the options are <tt>"l95"</tt> for the Lorenz 1995 model,
43 or <tt>"qg"</tt> for the QG model.
44 
45 The build examines the environment variable "ARCH" to determine which compilers

60 \section how_to_run How to run the toy system
61 
62 (In the following,
63 set <tt>model="l95"</tt> for the Lorenz 1995 model, or <tt>model="qg"</tt>
64 for the QG model.)
65 
66 There are three executables for each model. To run an assimilation, you
67 will typically need to run the following four steps:
68 
69  - Generate truth:
70  - <tt>${model}/${model}_forecast.x ${model}/scripts/truth.xml</tt>
71  - Generate observations from the truth:
72  - <tt>${model}/${model}_makeobs.x ${model}/scripts/makeobs4d.xml</tt>
73  - Perform a short forecast from the truth run, to generate the background:
74  - <tt>${model}/${model}_forecast.x ${model}/scripts/forecast.xml</tt>
75  - Run incremental 4dVar:
76  - <tt>${model}/${model}_4dvar.x ${model}/scripts/4dvar.xml</tt>
77 
78 Data files are saved in the "Data" directory, as specified in the XML files.
79 Make sure you have created the "Data" and "Data/FDB" directories before
80 you run the above.
81 
82 \section how_to_test How to test the toy system
83 
84 Executing the script <tt>$model/runtest.ksh</tt> will run several of the main
85 configurations and compare some of the key outputs to a control output stored in
86 <tt>$model/control.summary.txt</tt>.
