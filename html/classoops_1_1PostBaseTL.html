Detailed Description

template<typename INCR>
class oops::PostBaseTL< INCR >

Handles post-processing of model fields related to cost function.

PostBaseTL is the base class for all cost function processors, it is mostly used so that PostProcessorTL can hold a vector of such processors.

PostProcessorTL is typically attached to LinearModel::forecastTL method.

By default processing is performed on every call.

PostBaseTL is templated on INCR; INCR would typically be Increment.

Post-processing is organized in three stages/methods: initializeTL (called prior to time-stepping), processTL (called during forecastTL at each time step) and finalizeTL (called once the time-stepping is finished).

Methods initializeTL, processTL and finalizeTL wrap around virtual methods (implemented by PostBaseTL children classes) doInitializeTL, doProcessingTL and doFinalizeTL.

PostBaseTL class contains a PostTimer object. PostTimer class controls the frequency of post-processing.

PostBaseTL is the base class of the following classes (refer to their documentation for more information):

  • ObserverTL
  • WeightedDiffTL
  • JqTermTL
