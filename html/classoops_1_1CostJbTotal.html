Detailed Description

template<typename MODEL>
class oops::CostJbTotal< MODEL >

Jb component of the cost function.

CostJbTotal encapsulates B matrices corresponding to various components of the control increment/control vector:

Consider applying the matrix vector product:

\[ B dx = \begin{bmatrix} B_{state} & 0 & 0 \\ 0 & B_{modaux} & 0 \\ 0 & 0 & B_{obsaux} \end{bmatrix} \begin{bmatrix} dx_{state} \\ dx_{modaux} \\ dx_{obsaux} \\ \end{bmatrix} \]

  • CostJbState - \( B_{state} \): abstract base class encapsulating the background covariance matrix corresponding to the state component of the control increment. Concrete classes deriving from it are CostJb3D (3/4D-Var), CostJb4D (4D-Ens-Var), CostJbJq (weak constraint)
  • ModelAuxCovariance - \( B_{modaux} \): class encapsulating the background covariance matrix of an auxiliary model space component of the control increment (e.g.Q matrix - model bias covariance)
  • ObsAuxCovariances - \( B_{obsaux} \): class encapsulating background covariance matrices of auxiliary observation space components of the control increment (e.g. VarBC, TOVSCV)

Construct \( J_b\).

Set the pointer to the covariance matrix for the state component of the control vector. Construct the covariance matrices for the auxiliary observation and model space components of the control vector.

The covariance matrix for the state component of the control vector is constructed outside by calling the newJb method of one of the cost function classes derived from the abstract base class CostFunction: CostFct3DVar (returns CostJb3D), CostFct4DVar (returns CostJb3D), CostFct4DEnsVar (returns CostJb4D), CostFctWeak (returns CostJbJq).

Various computations after the integration of the model.

This method is called after the non-linear model integration. Its purpose is to compute the increment and evaluate the Jb cost function term value. For weak constraint the model error \( x_i - {\cal M}(x_{i-1}\) is computed. Returns the value of the Jb component of the cost function.

1.1 Compute x_0 - x_b for Jb

1.2 Compute model and obs space increments

  1. Compute model error \( x_i - {\cal M}(x_{i-1}\) (weak constraint only)
  2. Write increment (diagnostics)
  3. Write the model space component of the first guess
  4. Reset first guess pointer
  5. Compute Jb value

Various computations after the linearisation trajectory computation.

This method is called once the model linearisation trajectory is computed. Its purpose is to compute the first guess increment, update background covariances corresponding to the auxiliary observation space components of the control vector (relevant for IFS). In case of weak constraint JqTerm computes the model error. Returns the value of the Jb component of the cost function.

  1. Compute and save the first guess increment.
  2. Compute the model error (relevant for weak constraint)
  3. Update covariances corresponding to the auxiliary observation space components of the control vector
  4. Compute and return the Jb value
