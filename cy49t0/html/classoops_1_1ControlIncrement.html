Detailed Description

template<typename MODEL>
class oops::ControlIncrement< MODEL >

Control Increment.

The control variable acts as a container for the inputs of the variational data assimilation cost functions in physical space. That includes the states at the start the assimilation window or of each sub-window but also additional variables such as model bias, VarBC coefficients, or other control variables for algorithms that use them. The control variable increment contains variations of the control variable.

The control increment \( dx \) has the following structure

\[ dx = \begin{bmatrix} dx_{state4D} \\ dx_{modaux} \\ dx_{obsaux} \end{bmatrix} \]

where

\[ dx_{state4D} = \begin{bmatrix} dx^0 \\ dx^1 \\ .. \\ dx^N \end{bmatrix} \] 

and \f$ N \f$ is the number of sub-windows.

 ->  \f$ dx_{state4D} \f$ is implemented by Increment4D containing a vector
    of Increment objects

\f[  dx_{modaux} = \begin{bmatrix} dx_{modaux}^0 \end{bmatrix} \f]

 ->  \f$ dx_{modaux} \f$ is implemented by ModelAuxIncrement

\f[
     dx_{obsaux} = \begin{bmatrix} dx_{obsaux}^0 \\ dx_{obsaux}^1 \\
                                    ..           \\ dx_{obsaux}^K
                   \end{bmatrix}
\f]

 ->  \f$ dx_{obsaux} \f$ is implemented by ObsAuxIncrements which contains
    a vector of ObsAuxIncrement objects (\f$ dx_{obsaux}^i \f$), which in
    turn contain a map of ObsAuxIncrementBase objects
    (e.g \f$ dx_{VarBC}, dx_{TOVSCV} \f$):

 \f[
   dx_{obsaux}^i = \begin{bmatrix} dx_{obsaux,i}^0 \\
                            ..  \\ dx_{obsaux,i}^L
                  \end{bmatrix}
 \f]

 ->  IFS: K=0, L=1:

 \f[
   dx_{obsaux}= \begin{bmatrix}
                 \begin{bmatrix}
                  dx_{VarBC} \\ dx_{TOVSCV}
                 \end{bmatrix}
                \end{bmatrix}
 \f]
