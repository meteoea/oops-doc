Detailed Description

template<typename FLDS>
class oops::PostProcessor< FLDS >

Control model post processing.

Controls model post processing

This class controls model post processing in the most general sense, ie all diagnostics computations that do not affect the model integration. It contains a vector of PostBase objects. It just calls all the individual processors one by one.

Post-processing is organized in three stages/methods: initialize (called prior to time-stepping), process (called during the forecast at each time step) and finalize (called once the time-stepping is finished).

Methods initialize, process and finalize simply call the initialize, process and finalize methods, respectively, of the enrolled individual processors.

See the documentation of PostBase for the details of implementation of processors and the list of available processors.
